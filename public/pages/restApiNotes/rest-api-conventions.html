REST

What is REST?

REST, or Representational State Transfer, is a way of designing web APIs. It uses the resources it handles in the URL, uses standard HTTP methods to perform actions on or with the resources, and usually exchanges data between server and client in JSON format.

REST Conventions

Resource based URLs

We want our resources to be represented in our URLs. They should be plural, and when referring to a specific resource, we indicate it with an id.

Example endpoints:

  • https://my-movie-theater/movies — shows all movies.
  • https://my-movie-theater/movies/32 — shows the movie with id 32.

Status Codes

HTTP Status Codes represent the outcome of the action performed. The main ones we use are:

  • 2xx — success (e.g., 200 OK, 201 Created when creating a resource, 204 No Content when deleting a resource).
  • 3xx — redirection.
  • 4xx — client error (e.g., 400 Bad Request for invalid or empty JSON payloads, 404 Not Found when accessing a non-existent endpoint).
  • 5xx — server error.
