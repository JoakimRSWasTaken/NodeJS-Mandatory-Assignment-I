REST API Verbs

The Verbs

REST APIs use HTTP methods to define actions on resources:

Verb Purpose Description
GET Read Retrieves data from the server.
POST Create Sends data to the server to create new resources.
PUT Update (replace resource) Replaces an entire resource.
PATCH Update (replace fields) Updates specific fields of a resource.
DELETE Delete Removes a resource.

Requests

We use the REST verbs like GET and POST with Express to handle different kinds of requests to and from the server.

In the examples I will show some routes based on another project about movies. Note that this is nothing fancy and that all the movie objects are stored locally in an array. The req variable mentioned in the routes is the request sent to the server and the res variable is the response from the server to the client.

GET Requests

GET requests are used to retrieve data from the server. The things we GET from the server are called resources. A function in the app.js or in a Router is called a route.

GET all

A route that returns all resources.

app.get('/movies', (req, res) => {
    res.send({ data: movies });
});

Here we GET all movies stored by us when accessing the /movies endpoint.

GET by ID

A route that returns a specific resource.

app.get('/movies/:id', (req, res) => {
    const providedMovieId = Number(req.params.id);
    const foundMovie = movies.find((movie) => movie.id === providedMovieId);

    if (!foundMovie) {
        res.status(404).send({ errorMessage: `No movie found by id: ${req.params.id}` });
    } else {
        res.send({ data: foundMovie });
    }
});

This includes error handling. We cast the id from the URL to a Number and use req.params. If no movie is found, a 404 error is sent.

GET with Query Parameters

app.get('/bag', (req, res) => {
    // the query string starts with ? and is followed by key-value pairs:
    // bag?item1=durum&item2=burger
    res.send({ data: req.query });
});

Example query: bag?lunch=durum&dinner=burger

POST Requests

POST requests are used to create new data.

    if (!req.body) {  
        return res.status(400).send({ errorMessage: 'No JSON body provided.' });  
    }

{
    "title": "Jagten",
    "releaseYear": 2012
}

PUT Requests

PUT requests are used to update an entire resource.

app.put('/movies/:id', (req, res) => {
    const providedMovieId = parseInt(req.params.id);
    const foundMovieId = movies.findIndex(movie => movie.id === providedMovieId);

    if (foundMovieId === -1) {
        return res.status(404).json({ message: "Movie not found." });
    };

    let updatedMovie = {
        id: providedMovieId,
        title: req.body.title,
        releaseYear: req.body.releaseYear
    };

    movies[foundMovieId] = updatedMovie;

    res.send({ data: updatedMovie });
});

PATCH Requests

PATCH requests update only the provided fields of a resource.

app.patch('/movies/:id', (req, res) => {
    const providedMovieId = Number(req.params.id);
    const foundMovieIndex = movies.findIndex(movie => movie.id === providedMovieId);
    if (foundMovieIndex === -1) {
        return res.status(404).json({ message: "Movie not found." });
    };

    const foundMovie = movies[foundMovieIndex];
    const providedMovie = req.body;

    const updatedMovie = {
        ...foundMovie,
        ...providedMovie,
        id: providedMovieId
    };
    res.send({ data: updatedMovie });
});

DELETE Requests

DELETE requests remove a resource.

app.delete('/movies/:id', (req, res) => {
    const movieId = parseInt(req.params.id);
    const moviesArrayIndexToDelete = movieId - 1;

    if (moviesArrayIndexToDelete === -1) {
        return res.status(404).json({ message: "Movie not found." });
    };

    movies.splice(moviesArrayIndexToDelete, 1);
    res.status(204).send();
});

Movies are stored in an array, so deletion uses splice. The variable moviesArrayIndexToDelete accounts for array indexing starting at 0.
