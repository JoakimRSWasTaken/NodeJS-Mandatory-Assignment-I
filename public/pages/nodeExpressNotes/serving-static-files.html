Static Files

What are Static Files?

Static files are the files we serve to the client. This includes html, css, js, images, or similar. We keep the static files in a folder called public in the root folder: 

Example_Project/
├── node_modules/
├── public/
│   ├── assets/
│   │   ├── css/
│   │   ├── images/
│   │   └── js/
│   ├── components/
│   │   ├── footer/
│   │   └── header/
│   └── pages/
│       ├── about/
│       ├── contact/
│       └── frontend/
├── routers/
│   ├── pagesRouter.js
│   └── somethingElseRouter.js
├── util/
│   ├── pagesUtil.js
│   ├── navigationUtil.js
│   └── templatingEngine.js
├── app.js
├── nodemon.json
├── package-lock.json
└── package.json

What to do first?

We want to tell Express to serve all files in the public folder as static files. We do this with the following line of code: 

app.use(express.static('public'));

What to do next?

We send the static files with res.sendFile() as mentioned in the section about XSS. 

app.get('/', (req, res) => {
    res.sendFile(__dirname + "/index.html");
});

We use __dirname to get an absolute path to the file.

We can also use path.resolve(), which we will need when discussing ES Modules, from Node's own path module to resolve our paths. 

app.get('/', (req, res) => {
    res.sendFile(path.resolve('public/index.html'));
});
