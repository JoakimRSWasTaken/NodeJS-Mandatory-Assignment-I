Creating a Server with Express.js

What is Express.js?

Express.js is a web application framework for Node.js that is lightweight, modular, and unopinionated (to a certain degree).

You can read more about Express.js on their website if you are interested.

The three things to remember

To use Express to get a server up and running, there are three main things you want to do. You want to do all these things in your app.js. Once you have these three things down you are good to go, and the rest will follow!

1. Import

You want to import Express to access the libraries it contains. There are two main ways to do this; one is shown here, but note that there are two methods. 

const express = require('express');

You import Express as a variable with the require keyword.

2. Instantiate

You also want to instantiate Express, which can be done like this: 

const app = express();

Now you have instantiated Express.

3. Listen

Last, but not least, you have to make it listen on a port. An obvious choice for a port is 8080 since it is a standard choice for development purposes. 

app.listen(8080);

You want this to be at the bottom of your app.js. A lot of other developers expect it to be there, so it is more of an unwritten rule than a strict convention.

If you want to fancy up your app.listen(), you can do something like this: 

const PORT = process.env.PORT || 8080;

const server = app.listen(PORT, () => {
    console.log('Server is listening on port', PORT);
});

Here we define a variable called PORT and set it to be whatever we define it to be or 8080. Then we pass the PORT variable as an argument into the listen() function along with a callback function that logs a message to the console.
