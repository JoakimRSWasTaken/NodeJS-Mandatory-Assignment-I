Import and Export in Node.js

CommonJS vs. ES Modules - What are these?

CommonJS and ES Modules are two module systems supported by Node.js.

CommonJS

CommonJS is older and is the default in Node.js. It imports with require() and exports with module.exports. 

const cookie = require('./cookie.json');
  
function getCookie() {
    return cookie;
}
module.exports = getCookie;

ES Modules

ES Modules are a more modern JavaScript module system. It uses the keywords import and export. It supports named imports and default imports. ES Modules also requires .js in imports, but it is recommended to always include it for consistency and readability. 

import express from 'express';
import { getNewCookieBatch } from './cookieFactory.js';

export default function getNewCookieBatch() {
    return ['cookie1', 'cookie2'];
}

Here we import a default export, express, and a named export, getNewCookieBatch.

Main take-aways

You cannot mix require() and import in the same file. __dirname is not available in ES Modules, which is why we use the static library from Express.js.
