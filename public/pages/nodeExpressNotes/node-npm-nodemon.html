Node.js, npm and nodemon

What is Node.js?

To explain why Node.js is cool we have to mention how vanilla JavaScript works. JavaScript is typically run in the browser. Node.js is a JavaScript runtime environment that lets you run JavaScript on the server side. This allows us to interact with the server itself and with data storage using JavaScript. This feature allows us to do many cool and awesome things, including but not limited to server-side rendering of webpages.

Node Modules

Node Modules are JavaScript libraries of code that can be installed with npm and imported into your project. Node Modules are stored in a folder called node_modules in your project.

What is Node Package Manager?

Node Package Manager, or npm for short, is a package manager for Node.js. It makes sense. It allows you to install and manage dependencies for your Node.js project. Using npm is pretty easy and we will need it for what we want to do.

npm install

npm install is the command you want to use when installing a new package. An example would be to use it to install Express, which we will talk about in another section. 

$ npm install express

The $ signifies that this is a command typed into your terminal shell. You would want to install Express in the root folder of your project.

What is nodemon?

nodemon is a package you can install with npm just like Express is. nodemon lets you automatically restart your application whenever it detects changes in your files. This is amazing for development purposes as it saves you the trouble of manually stopping and restarting your application.

To run your application with nodemon you want to combine the command with the JavaScript file you want to run: 

$ nodemon app.js

nodemon.json

nodemon.json is a file that lets you configure how nodemon works for your project. The main thing I will point out for this is that nodemon by default will listen on the file types .js, .mjs, .cjs and .json. Since we will be working with .html and .css as well, we would like to add those to the list. To do this we would create the nodemon.json file in our project root folder and add the following into it: 

{
    "ext": "js, json, html, css"
}

This overwrites the list of file types that your nodemon listens in on, so add all file types you would need.
