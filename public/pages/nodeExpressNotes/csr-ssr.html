CSR vs. SSR

What do these letters mean?

I understand your confusion. You are not alone.

CSR

CSR, or Client-side Rendering, is a way to render your website. More accurately, it allows the client to render the content. HTML files are rather empty when using CSR and get filled out by JavaScript as the page loads.

This can be good for dynamic pages but may feel slow on less powerful devices. Historically, it was poor for SEO (Search Engine Optimization), since web scrapers used to only scan HTML content. Empty HTML would lead to missing content for your page. Modern web scrapers now execute JavaScript, so this is less of an issue, but it’s worth keeping in mind.

SSR

SSR, or Server-side Rendering, is another way to render your website. Here, the server renders the HTML fully before sending it to the client.

This is beneficial for SEO, as the full HTML content is immediately available for web scrapers. It also reduces client workload by offloading rendering to the server.
