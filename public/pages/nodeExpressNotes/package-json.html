package.json

What is the package.json file?

package.json is the file that contains the metadata about your project. If you have worked with Maven as a build tool before, you might be familiar with the pom.xml which fulfills the same purpose. 

Maven               NPM

=================================================

pom.xml             package.json

meta                meta

dependencies        dependencies

lifecycles          scripts

A lot of the values to be set in package.json have default values, meaning you don't have to fill them out if you do not want to. Here is an example of a well-defined package.json: 

{
  "name": "example_project",
  "version": "0.0.1",
  "description": "An example of a package.json",
  "keywords": [
    "example",
    "package.json"
  ],
  "license": "ISC",
  "author": "JRSWT",
  "type": "module",
  "main": "app.js",
  "scripts": {
    "start-dev": "cross-env PORT=8080 NODE_ENV=DEV node app.js",
    "start-prod": "cross-env PORT=80 node app.js"
  },
  "dependencies": {
    "cross-env": "^10.1.0",
    "express": "^5.2.1"
  }
}

And here is an example of a still completely viable package.json: 

{
    "dependencies": {
        "express": "^5.2.1"
    }
}

However, if you plan on releasing your project as a package, it is recommended to fill it out to look more like the first example.

dependencies

Dependencies is a list of libraries, or modules, that your project is using. They will be listed with their name and a version, e.g. 

{
    "dependencies": {
        "express": "^5.2.1"
    }
}

The ^ used in the version tells npm to install the latest version that is greater than or equal to the given version.

scripts

Scripts are terminal commands that can be executed with the npm run command. Remember the fleshed-out package.json from earlier? It had a script called start-dev and that script can be run like this: 

npm run start-dev

This is handy for defining specific ways to run your app or similar tasks.
