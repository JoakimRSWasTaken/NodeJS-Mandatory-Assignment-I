Date in JavaScript

We are not going on a date but we will be exploring how to work with time and date in JavaScript.

Local Time


console.log(new Date().toString()); // Local Time

This line logs the local time, which may not match the UTC time due to time zone differences.

Unix Epoch (Milliseconds since midnight 1/1 1970)


console.log(Date.now()) // Unix Epoch (Milliseconds since midnight 1/1 1970)

The Date.now() method returns the number of milliseconds since January 1, 1970, 00:00:00 UTC. This is often used to represent the number of seconds since the Unix epoch.

Internationalization

JavaScript's Date object provides methods for internationalizing dates, such as toLocaleString(). In the app.js file, we're using this method to get the current month and day in the Danish language (da-DK). 


app.get('/months/v1', (req, res) => {
  const currentMonth = new Date().getMonth();

  res.send({ data: months[currentMonth] });
});

app.get('/months/v2', (req, res) => {
  const currentMonth = new Date().toLocaleString('da-DK', { month: 'long' });

  res.send({ data: currentMonth });
});


app.get('/days/v1', (req, res) => {
  const weekday = new Date().getDay();

  res.send({ data: days[weekday] });
});

app.get('/days/v2', (req, res) => {
  const weekday = new Date().toLocaleString('da-DK', { weekday: 'long' });

  res.send({ data: weekday });
});

In these examples, we're using the toLocaleString() method to get the current month and day in Danish, respectively. The options { month: 'long' } and { weekday: 'long' } specify that we want the full month name and the full weekday name.
