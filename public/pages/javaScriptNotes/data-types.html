Data Types

Which data types do we have in JavaScript?

We have 8 different data types.

  • Boolean
  • Symbol
  • Number
  • BigInt
  • String
  • Object
  • null
  • undefined

Boolean

The boolean data type represents either true or false. 


isLearningJavaScript = true;

Number

The Number data type represents real numbers. This means both integers and decimals are included. These are widely used, as you can probably imagine. 


const heightInCentimeters = 188;
const weightInKilograms = 76.5;

BigInt

The BigInt data type also represents a number, but it is usually reserved for large integers. 


const largeInteger = BigInt(1234567890987654321);
console.log(largeInteger.toString());

With the above statement the console will log the string 


"12345678909876543210000000000"

String

The String data type represents a sequence of characters that can be both letters, numbers and symbols. There are several ways to define strings in JavaScript: 


const quotationMarks = "These are ";
const apostrophes = 'the 3 ways to ';
const backTicks = `define strings in `;

You can do operations on strings, like concatenation. 


const definingStrings = quotationMarks + apostrophes + backTicks;
console.log(definingStrings);

The above statement will log the following string to the console: 


"These are the 3 ways to define strings in "

You can also use backticks to have the values of variables displayed in your strings: 


const js = 'JavaScript!';
const definingStringsInJavaScript =
`${quotationMarks.trim()} ${apostrophes.trim()} ${backTicks.trim()} ${js}`;

The definingStringsInJavaScript variable would then produce the string 


"These are the 3 ways to define strings in JavaScript!"

Another thing to note about the different ways to define strings is that they allow you to use the other defining characters in your string should you need them. For example: 


const quoteFromSomeone =
'They said "It was the best of times, it was the worst of times" if I remember correctly.';

This example shows how to use apostrophes to define the string, enabling the use of quotation marks within the string itself.

Object

The Object data type allows you to store several lines of data within a single object: 


const thisSong = {
  isTheBestSongInTheWorld: true,
  description: 'The best song in the world.'
}

We can both add, remove and change properties to our objects: 


// Adding a new property
thisSong.about = 'This is just a tribute.'

// Removing a property
delete thisSong.description;

// Changing a property
thisSong.isTheBestSongInTheWorld = false;

Our object would now look like this if logged in the console: 


{
  isTheBestSongInTheWorld: false,
  about: 'This is just a tribute.'
}

Symbol

The Symbol data type creates an immutable and unique object-like value. 


const key1 = Symbol('key1');

Avoid thinking too hard about this one as it will not be mentioned again.

null

The null value represents the absence of a value. It is not 0 and it is not an empty string ('') as those are actual values.

This can be useful if you were getting a value from somewhere and would like to handle the case where something went wrong and the value was missing. 


if (thisValue === null) {
  console.log('No value here');
} else {
  // do something else
}

undefined

The undefined data type represents an uninitialized variable or property. It is also the return type of some functions that do not return anything, like console.log().

Running the following in your Node REPL: 


console.log(2 + 2)

would print these two lines for you: 


4
undefined

The function prints the result to the console and then returns undefined.
